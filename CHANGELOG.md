# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/), and this project follows [Semantic Versioning](https://semver.org/).

## [Unreleased]

### Added

- **`read_instructions` / `update_instructions` model tools.** The model can now read and manage Persistent Instructions through two universal host tools on every non-Stage Engine. `read_instructions` (an observation) returns one exact `{ scope, engine }` target's content, size, hash, and whether it is selected for the active Engine. `update_instructions` (a mutation) writes or deletes one target through the instruction store: atomic write (temp + rename), optional optimistic concurrency via `ifMatchSha256` (`"absent"` or a 64-hex hash; a stale value never overwrites), a single recoverable previous-state backup under `.vesicle/instruction-backups/` (project scope) or beside `providers.yaml` (user scope), and a pre-write check that the new content plus the other scope stays within the 32 KiB budget for every Engine the change affects. Empty content creates an explicit empty override; deleting restores fallback. Targets are a fixed `{ scope, engine }` enum — no arbitrary path — and live outside the model-visible writable roots, so this is a bounded host exception, not a widened filesystem surface. `update_instructions` routes through the existing Tool Permission Runtime: MANUAL and INERTIA pause via the standard permission request, MOMENTUM and YOLO execute immediately. A successful update refreshes the in-turn frozen instruction snapshot, so it takes effect on the next provider round of the same turn. Stage remains strictly tool-less by design. A custom unified-diff permission preview is deferred.
- **`/init [notes]` project initialization.** Scans the project's writable roots (`workspace/`, `source_materials/`, `novels/`, `reports/`, `test_runs/`) and makes one no-tools provider call with a dedicated host prompt to draft a project-scope `VESICLE.md` — the same Persistent Instructions file the host auto-loads every session. It is the companion to Persistent Instructions: instead of hand-writing the first `VESICLE.md`, `/init` seeds a concise draft (project overview, recommended engine, character/scenario conventions, workflow and naming conventions, non-obvious gotchas) that takes effect on the next turn and that you then review and edit. The init prompt lives in the Vesicle host-assets layer, so it needs no new Harness from Neural-Narratology. An existing `VESICLE.md` is backed up to `.vesicle/init-backups/VESICLE.md.previous` before being replaced, never silently overwritten. `/init` runs as a cancellable host action (like `/compact`), independent of the active Engine's workflow.
- **Persistent Instructions.** User-authored Markdown that customizes an Engine's workflow and survives new sessions — the host loads it into the system prompt automatically, so there is no need to ask the model to write a spec to a file and remind it to read it next session. Files live at two scopes, both using the same names: `VESICLE.md` (general, every Engine) and `VESICLE.<engine>.md` (Engine-specific override) at the project root for project scope, and the same names beside `providers.yaml` for user scope, which applies across every project root. Within one scope an Engine-specific file replaces the general file; across scopes the user file is followed by the project file, and project content wins on a direct conflict. Instructions are appended after the byte-identical Engine prompt as host context — the Engine contract stays the single system authority and the stable prefix-cache prefix — and can only customize work within existing host capabilities; they cannot add tools, permissions, gates, validators, or filesystem authority. Selection is resolved from current disk when a top-level turn begins, when a session resumes after a process restart, and on a confirmed Engine switch; within a single turn the selection is frozen so a mid-turn pause and resume observes one stable instruction set, and an edit takes effect on the next turn. An invalid, linked, or oversized scope is skipped with a warning while the turn continues, and the combined selected content is bounded to 32 KiB. `vesicle prompt shape` / `prompt dump --engine <id>` and the `/instructions` command show the effective selection, provenance, byte budget, and diagnostics.
- **`vesicle --version` / `-v` and `--help` / `-h`.** A typed startup parser now classifies one invocation into a terminal action (`--version`/`-v`, `--help`/`-h`), a subcommand, or a project launch. `--version` prints the package version baked into the binary at build time; `--help` prints the global usage. Short options bundle (`-vh`), `--` ends option parsing so dash-prefixed paths work, and `--dangerously-skip-permissions` remains a position-agnostic process flag. On source/npm launches Bun's runtime consumes a leading `--`, so use `vesicle launch ./-path` there; the compiled binary preserves it.
- **`vesicle --resume` / `-r` [path].** A launch modifier that opens the session picker on startup, routing to the same host-owned picker as `/resume` with no argument — opening the picker is state-only and never starts a provider turn until a session is chosen. `-r` bundles with other boolean short options; `--resume` is rejected for subcommands and terminal actions.
- **Message and command queue.** While the Agent Loop is running, Enter queues ordinary text and image input, shows a bounded mixed FIFO preview above the composer, and clears the draft for another input. Queued messages steer the active loop after its current complete tool round and before the next provider request. Every slash command now declares a required busy-turn behavior in the command registry: safe host-only commands run immediately, artifact reads wait for the current tool round, and configuration, picker, or session commands wait for the Agent Loop. Escape interrupts the current provider or tool operation and processes the next queued input immediately. Empty-draft Up retrieves the latest queued input for editing.
- **`/btw <question>` side questions.** Ask one tool-free question about the current conversation without interrupting the active turn. `/btw` copies the frozen provider-valid context boundary published before each main provider request, so it never observes a half-written tool round, then projects it into a single request with exactly one system authority (the dedicated side-question prompt) and one user message that quotes the parent Engine prompt, the conversation, and tool results as inert reference data — parent workflow intent, tool protocol, and reasoning state never become active side instructions. No tools are declared or executed; any structured tool call in the response (including mixed text-plus-tool) fails the exchange. The request runs under a side-specific AbortController independent of the main turn, and the answer streams into a full-area ephemeral overlay that reports main status while the turn continues underneath; bare `/btw` reopens the latest in-memory answer and `←/→` navigate, `c` copies, `↑/↓` scroll, and `x` clears the current session's exchanges. Side exchanges never enter session JSONL, the main transcript, checkpoints, validators, gates, permissions, or tool execution, and do not survive a process restart.

### Fixed

- **Module A/B validator false positives on non-card content.** The card validators' applicability test treated any assistant response starting with `---` (a Markdown horizontal rule) as a YAML-frontmatter artifact, so a plain report or phase-transition note was validated as both a character card and a scenario card and surfaced a full batch of spurious "missing section / missing field" findings. Card applicability now recognizes real frontmatter by key family and Module A body sections, so a `---`-led report triggers no card validator. The turn-finalizer and `/validate` now run only the validators whose `applies` predicate matches (a character card is no longer cross-validated as a scenario card and vice versa) through one shared path, and classification no longer depends on the very fields a validator is meant to diagnose (a card missing `archetype` or `scenario_name` is still recognized).
- **Assets CLI errors** now exit with a concise user-facing message instead of leaking a Bun stack trace when Harness manifest or argument validation fails.

## [1.0.0-alpha.3] - 2026-07-21

### Added

- **Stage consumer Engine.** The first-party `/stage <character-card-path> <scenario-card-path>` freezes Module A and Module B content unchanged, renders and persists the opening bootstrap context, and resumes that frozen context even when the source cards later drift. Stage exposes no model-visible tools, gates, MCP, Agent, or shell surface; it validates the shared three-part packet and records `stage.prose` observations. The player view keeps prose primary with a compact HUD.
- **Experimental Runtime Semantic Judge.** Opt-in via user-level `quality.yaml` (defaults to `off`); `rewrite` requires an explicit confirmation command. The host records only a secret-free profile snapshot and bounded results, preserves strict Judge parsing with one repair, and rejects pending retry after profile drift. Separate from calibrated Policy activation; makes no production-quality or AI-authorship claim.
- **Developer `vesicle quality benchmark` command.** Runs an explicit provider/model matrix from a frozen plan, requires `--allow-live`, appends resumable hash-only rows, and writes per-model Wilson/slice reports without candidate text or raw responses. Records measurement evidence only; cannot enable semantic blocking. Plans may set `earlyStop.minimumEvaluations` to avoid applying a rate-based early stop before an operational pilot has enough observations.
- **`quality-policy/semantic-rewrite@1` recognition.** A Harness that requires it must publish an active, hash-verified policy with stable Judge rules, exact protocol/model scopes, complete per-rule confidence thresholds, and non-placeholder calibration digests. The host exposes pure eligibility evaluation only; published Harnesses remain observe-only and semantic findings still cannot enter rewrite decisions.
- **Pinned Biome correctness linting** for TypeScript/TSX source, scripts, and tests, enforced by the reusable CI/release build before typechecking, with an opt-in tracked pre-push hook that blocks pushes with lint diagnostics. Formatting and import assists stay disabled; Harness assets stay outside the lint scope.

### Changed

- **`STATUS.md` rewritten as an implementation-inventory index** — grouped capability-state table, classified limits list, refreshed repository tree, and links into the authoritative `docs/dev/` documents. No runtime behavior changed.
- **Windows Authenticode signing deferred** with no version deadline. Artifacts remain unsigned and rely on the GitHub Release, SHA-256 checksums, and npm provenance; the earlier `1.0.0-beta.1` signing deadline is superseded with no replacement gate. `CODE_SIGNING_POLICY*`, `README*`, `STATUS.md`, `CONTRIBUTING*`, and the user-manual signing reference were updated to match.
- **User manual rewritten** from a single linear Windows-first curriculum into a channel-funnelled structure: a routing page, four start pages (Windows installer, npm, Windows portable, Linux portable), five channel-agnostic tutorials, and a reference section, grounded in the current CLI, guided Setup wizard, doctor output, validators, config files, installer, and release artifacts.
- **User-manual language policy flipped to Simplified-Chinese-canonical** with an English mirror at the same relative paths; root and `docs/dev` governance docs remain English-canonical.
- **npm path recommends a global install** (`npm install -g prism-vesicle`) so the standard workflow is `cd project && vesicle .`.
- **New `advanced/` user-manual section** (host shell/Process Runtime, Output Quality Guard, SubAgents, Stage) with Simplified-Chinese canonical and an English mirror. Each page marks a point-in-time 🟢/🟡 status that defers to STATUS.md, so experimental features can graduate without rewriting the prose.
- **Bundled V10 baseline refreshed** to Neural Narratology Release `harness-20260720-3` / `prism-engine-v10@10.1.0-rc.1` (commit `90f65c9`): exact 73-file manifest inventory, profile-declared host briefs, compact quality projections, raw Stage bootstrap template, and static prompt-asset ledger.

### Fixed

- **Stage transcript projection** folds blank lines around the concealed Neural Chain and compact HUD without stripping meaningful indentation from the visible prose, while preserving one stable visual gap below the HUD.
- **Stage source-view toggle** uses `Ctrl+Alt+S` instead of plain Enter or Space, so a focused Stage message cannot intercept composer text, command spaces, prompt submission, or `Ctrl+Enter` newlines.
- **Output Quality Guard artifact enforcement** derives targets only from successful `create_file`, `write_file`, `replace_in_file`, and `append_file` results, reads each guarded path's complete current post-image at the quality boundary, and keeps blocking paths pending independently across rewrites, pauses, cancellation, and restart. A clean completion summary or unrelated clean file can no longer make an unchanged bad artifact pass.
- **Quality Guard exhaustion** no longer falls through as ordinary completion. Quality events separate assessment outcome from host action; append-only warning and resolution records preserve exhausted, unreadable, and oversize targets across restart, and the TUI restores interrupted or exhausted revisions as a three-way decision (retry once with the same Engine / use the current version with its warning / stop without a provider call).
- **Semantic Judge benchmark** rejects unsupported corpus target types before calling a provider, and supports a bounded frozen per-evaluation timeout that participates in the plan hash without changing the interactive Runtime's 15-second deadline.
- **Guided Setup** bounds page descriptions and budgets model-selection rows from the actual compact or regular panel structure, preventing text overlap when Setup starts in a non-maximized Windows Terminal window.
- **Fixed-height TUI rows** opt out of implicit OpenTUI word wrapping; shared measurement, truncation, padding, and composer editing preserve grapheme clusters using terminal display columns instead of JavaScript string length. Rewind, Permission, and YOLO panels reserve bounded rows so current-point, warning, error, and approval controls stay visible.
- **Slash-command argument completion** is now command-owned instead of a controller whitelist: built-in commands register their own grammar and candidate source, and `/quality`, `/artifact`, `/validate`, `/resume`, `/stage`, and `/engine --summary` now complete their arguments.

## [1.0.0-alpha.2] - 2026-07-15

### Added

- **SubAgent runtime.** Profile-driven foreground/background child agents (`spawn_agent`, `list_agents`, `send_message`, `interrupt_agent`, `wait_agent`) with bundled Explore, Plan, Research, Reviewer, and General profiles and sparse project/user overrides under `assets/agents/`. Foreground children pause only the parent model loop while streaming progress; background children return immediately, run concurrently, persist completion in a durable parent inbox, and trigger an automatic parent continuation when the session is idle. `/agents` exposes status and cancellation.
- **First-class SubAgent observability** — dedicated lifecycle cards, live progress and bounded result previews, persistent active/ready summaries in the header and Workspace sidebar, background delivery states, restored cards on session resume, and `/agents <handle>` detail with argument completion.
- **Dual SubAgent identity** — opaque host-only `runId` for storage and recovery plus stable parent-scoped handles such as `explore-1` for model tools and user commands; legacy UUID references remain input-compatible.
- **Directory tools** — `list_directory`, `create_directory`, `move_directory`, and `delete_directory` (empty, non-root only). File checkpoints now preserve directory topology, parallel Agent write ownership detects ancestor/descendant conflicts, and model-visible project paths reject symbolic-link traversal.
- **Contract-bound Harness delegation** (`prism-agent/delegation@1`) over the existing SubAgent runtime. Verified Driver Contracts uniquely bind parent Engine, Agent Profile, execution mode, purpose, and retry limit; reject undeclared, ambiguous, or mode-escalating requests; serialize Harness delegations; normalize Driver ABI errors; persist attempt and terminal metadata; and open the declared resumable user decision point when transient retries are exhausted.
- **Fail-closed Prism Harness Pack foundation** — strict `prism-harness-pack/v1` parsing, exact file/hash and Profile/Prompt binding verification, Adapter/capability compatibility checks, external host asset validation, and staging-based immutable directory installation. `/permissions` remains the sole tool-call authorization layer rather than being duplicated in Harness or HAL.
- **Offline managed-Harness lifecycle** — `vesicle assets verify`, `install`, `use`, `status`, and `rollback` accept already-extracted packs, persist exact project and session identity, reverify immutable content on activation and resume, and select one complete managed baseline below sparse overrides. Missing managed files cannot fall through to bundled V10; rollback atomically restores the whole bundled V10 baseline.
- **Bundled V10 baseline** auto-activates when no project lock exists: exact Harness manifest inventory, root `harness-manifest.json`, and a restricted host extension layer. New sessions persist the Harness identity (pre-V10 sessions fail closed on resume) and a content-only fingerprint of the effective merged asset tree that warns on resume drift.
- **Output Quality Guard** (`quality-guard/anti-ai-flavor@1`) — validates the released Rule Pack and detector contracts, preserves normalized evidence offsets across protected Markdown/HUD regions, buffers Runtime prose until Guard policy resolves, requests at most two rewrites from the original Engine, stops on repeated candidate hashes, and persists resumable bounded QualityEvents. Dyad, Weaver, Weaver-Orch, and Scene Writer use observe paths; Evaluate and Chapter Reviewer reports are not recursively guarded.
- **Global asset overlays** — `vesicle assets status`, `materialize <assets/path> [--global]`, and `init --global` create user-wide or project-specific editable layers. Runtime assets resolve file-by-file through sparse project overrides, user-global overrides, one managed/bundled V10 baseline, and the restricted host layer; model-visible tools see only logical `assets/...` paths and never physical global or package locations.
- **Tool Permission Runtime** — four coarse modes: `MANUAL` asks for every tool, `INERTIA` auto-allows observation tools, `MOMENTUM` auto-allows all tools except `shell_exec`, and `YOLO` auto-allows the effective surface after two red confirmations. MCP tools are always treated as side-effecting, and SubAgent requests route through the parent TUI.
- **Opt-in non-interactive `shell_exec`** backed by a bounded cross-platform Process Runtime: fixed project cwd, filtered child environment, separate stdout/stderr limits, wall-clock timeout, process-tree termination, exact-plan approval hashes, durable request/resolution/process metadata, and no replay after indeterminate crash recovery.
- **Background shell execution** via `shell_exec.runInBackground` returns a managed `shell-N` task immediately, with bounded status/output persisted under `.vesicle/processes/`, next-turn completion notification, and `shell_output`/`shell_stop` controls.
- **Shell interpreter profiles** — host-owned `auto`, `powershell-7`, `windows-powershell-5.1`, `cmd`, `git-bash`, or Linux/WSL `posix-sh`. Windows `shell_exec` no longer requires PowerShell 7; the resolved executable and runtime policy bind into the approved plan, PowerShell/CMD output is normalized to UTF-8, model guidance follows the selected command dialect, and unavailable explicit profiles fail closed instead of crossing shell families.
- **User-level `permissions.yaml`, `/permissions`, and the process-scoped `--dangerously-skip-permissions` override.** Persistent YOLO defaults are refused and resumed YOLO sessions downgrade to MOMENTUM unless the dangerous CLI override is active.
- **Guided per-user Windows installer** (Inno Setup 6) with a stable upgrade identity, complete standalone V10 payload, Start Menu Setup/Doctor entries, exact user-PATH add/remove behavior, preserved user/project state, and Windows CI install/runtime/uninstall smoke coverage. PR CI and tag-triggered publishing carry the versioned installer alongside the portable artifacts.
- **Reusable CI/release build** consolidated onto one Linux/Windows pipeline that owns release gates and artifacts; PR and `develop` CI retain short-lived versioned PE, ELF, assets-ZIP, and installer artifacts for review.

### Changed

- **1.0.0-alpha.2 authorized as an explicitly disclosed unsigned Windows prerelease** for the informed alpha group while the SignPath Foundation application is pending. The GitHub Release prepends a bilingual unsigned-artifact warning, links the code-signing policy, and directs users to `SHA256SUMS.txt`; the temporary publication workflow pins the publishable package version to exactly `1.0.0-alpha.2` until a reviewed signing integration or another explicit alpha decision replaces it.

The normal path is command-line driven with no Actions-page dispatch, Candidate workflow, or GitHub Environment approval step; future signing approval remains a separate manual trust gate.
- **Agent-loop and code-smell refactors** split the provider loop and remaining P1 runtime modules into narrower responsibilities without changing external behavior.

### Fixed

- **Provider-returned tool calls** are checked against the current effective tool surface before permission evaluation or execution, so YOLO cannot execute a hallucinated `shell_exec` when the host-shell capability is disabled.
- **Permission recovery** fails closed across capability/config drift and interrupted multi-tool windows: resumed approvals cannot re-enable a disabled tool, incomplete calls are never replayed, and shell-tainted checkpoints surface a targeted rewind warning.
- **MANUAL and INERTIA approvals preserve the parallel foreground SubAgent contract** by collecting exact per-call decisions before starting the approved Agent batch concurrently; simultaneous parent/child prompts resolve the request actually shown.
- **Mixed host-tool and SubAgent rounds** persist every already-started SubAgent result before propagating a sibling host-tool failure, preserving the durable tool-call/result pairing for recovery.
- **Cancelling a background SubAgent** no longer enqueues a synthetic result or wakes the parent Engine for another provider turn; cancellation is a durable terminal Agent state, and legacy queued cancellation notices are acknowledged without delivery.
- **SubAgent lifecycle and progress events** no longer overwrite the parent Engine's Workspace STATUS line; activity remains visible through dedicated cards, the header summary, the Agents sidebar, and activity records.
- **Interrupted foreground Agents** close their original `spawn_agent` tool call during crash recovery, background delivery blocks until provider/gate state is coherent, terminal children reject late control requests, and `/agents retry` explicitly resumes a delivery paused after exhausted provider retries.
- **Workspace sidebar** keeps a fixed two-row Shell summary, preventing multiple background tasks from overwriting the Effort, session, and MCP rows.
- **Permission pauses** no longer render empty assistant bubbles, and the active TUI bottom surface owns keyboard and paste input through the same modal priority used for rendering.
- **Shell process deadlines** remain active until inherited stdout/stderr pipes close, and successful shell exit also cleans up surviving in-group descendants instead of leaving ordinary child work behind.
- **Generic host SubAgents** (`explore`, `general`, `plan`, `research`, `reviewer`) remain ordinary concurrent host Agents while a V10 Harness is active, while undeclared non-host Agent Profiles still fail closed through the Driver Contract.
- **Guided Setup no longer persists one first-project directory** or routes Start Menu launches back to it. Project choice is optional and one-time, legacy `setup-state.json` pointers are ignored without deletion, invalid launch paths produce one bounded CLI error, and backward navigation no longer overlaps compact terminal layouts. Installed users receive the native `vesicle.exe` command plus the per-user Explorer action; upgrades remove legacy launchers, and rerunning the installer presents Reinstall/Repair/Uninstall.

## [1.0.0-alpha.1] - 2026-07-11

### Added

- **Thinking blocks and reasoning visibility** — provider-neutral internal thinking blocks with OpenAI-compatible `reasoning_content` as a compatibility bridge, preserved through streaming, tool-loop follow-ups, and session resume. `/effort off|low|medium|high|xhigh|max` controls runtime thinking effort (plus `/effort auto`/`unset` to clear), and `/reasoning hidden|collapsed|expanded` controls display with bounded collapsed/expanded tail views.

Models opt in with `capabilities.vision: true`; the TUI accepts clipboard images with `Alt+V` (and WSL-compatible `Ctrl+Alt+V`), keeps atomic `[Image #N]` elements through history/rewind/resume, and stores bytes in a content-addressed `.vesicle/attachments/` store with base64 only on the in-memory provider request copy.

Discovered tools expose as `mcp_<prefix>_<tool>` aliases, execute through `tools/call`, surface structured `mcpEvent` metadata, appear in prompt dumps, and are reported by `vesicle doctor` without printing secret header values. The Workspace sidebar shows a compact MCP status section.

- **Rewind and per-turn checkpoints** — `/rewind` (alias `/checkpoint`) and empty-input double Esc open one selector that can restore conversation, file checkpoints, both together, or summarize from a selected prompt, forking future turns while preserving abandoned branches in append-only session JSONL. Per-user-turn checkpoints live under `.vesicle/file-history/` with pre-mutation backups, changed-file and insertion/deletion previews, a 100-snapshot active limit, and branch-aware resume.
- **Engine switching** — `/engine [id]` for manual Prism engine inspection and switching, plus the `request_engine_switch` model-visible handoff tool that pauses for confirmation and switches only future turns after the user confirms. Both share one unified transition record; confirmed switches append a bounded user-role `engine_handoff` packet so OpenAI-compatible, Anthropic Messages, and Gemini adapters all receive the same target-engine context without mutating the dynamic system prompt.
- **`ask_user_question`** model-visible clarification tool — one single-select question with 2-4 model options plus host-owned Skip and open-ended answer fallbacks, keeping question-panel arrow-key selection from scrolling the message history and resuming the current engine loop after selection.
- **`/compact [notes]`** summarizes the active session through the configured provider and replaces old context with a user-role summary on a new append-only branch. `/engine <id> --summary [notes]` compacts first then switches; model-requested handoffs now also offer `Confirm with summary`.
- **Cross-provider usage normalization** — provider responses normalize runtime usage metadata across all three adapters. The TUI footer shows compact logical-turn upstream/downstream token totals, cached-input hits, and context-window percentage when configured, plus session totals that add logical-turn summaries instead of re-counting repeated context sends; sessions persist the underlying telemetry as host-only metadata. `/context` reports configured limits, latest occupancy, session totals, and auto-compact metadata without calling the provider.
- **Provider/model registry and artifact workbench** — a user-level `providers.yaml` declares multiple providers and models plus an optional `defaultModel`, and the TUI offers a two-step `/model` picker, provider-default shortcuts, and exact provider/model selection. `/artifact [n|path]` lists or previews generated files as bounded structure-preserving cards, and `/validate` checks the selected file on disk.
- **Provider cancellation** via `AbortSignal` for OpenAI-compatible, Anthropic Messages, and Gemini requests; Esc during generation interrupts the request and restores the submitted prompt for editing.
- **Slash-command candidate menu** — a filtered popup while typing a command token, with Up/Down or Ctrl+P/Ctrl+N selection, Tab/Enter completion, and Escape cancellation. `/model`, `/engine`, `/effort`, and `/reasoning` complete their arguments, including alias matching that completes to canonical values.
- **Standalone executables** — `bun run build:exe` cross-compiles a Windows PE (`prism-vesicle.exe`) and a host ELF (`prism-vesicle`) from WSL in one run, embedding OpenTUI's tree-sitter worker as a flat Bun worker entrypoint so no external `node_modules/` runtime bundle is needed. The compiled binary loads its immutable default `assets/` beside `process.execPath`. `vesicle debug markdown-runtime` verifies the worker, WASM runtime, and highlight probes without starting the TUI; `bun run build:assets` creates the separately distributed editable `dist/prism-vesicle-assets.zip` release pack.
- **GitHub Actions CI** validates Linux ELF and native Windows PE release shapes, including the standalone Markdown runtime diagnostic and external assets; pull-request runs upload short-lived versioned artifacts for review without publishing a GitHub Release or npm package.

- **Provider retry** — pre-response network failures, HTTP 408/429, and 5xx responses retry up to two times with bounded exponential backoff, jitter, and `Retry-After` support across all three adapters. Cancellation interrupts backoff immediately; partially consumed SSE streams are never replayed implicitly.
- **"Synaptic Prism" TUI rewrite (phases A–E)** — a deep cool surface with a single emerald prism accent; a per-role spectrum lane on each message with asymmetric containment; inline filesystem tool-call cards with real line-level diffs and structured outcome footers; a per-turn `▣ {engine}·{model}` marker with per-engine accent colors; a stream-first single-sidebar layout; live turn-phase status tracking; an `engineDisplayName` helper (etl → ETL, runtime → Runtime, weaver-orch → Weaver-Orch); and a `scripts/palette.ts` dev swatch utility.

Plain Enter submits, `Ctrl+Enter` inserts newlines, `Shift+Enter` is inert when reported distinctly, and trailing backslash+Enter remains a compatibility newline fallback.

### Changed

- **Shared Markdown display layer** conservatively converts common LaTeX formulas to terminal-readable Unicode and downgrades common Markdown extension syntax (`==mark==`, `~sub~` / `^sup^`, footnotes, definition lists, image alt text, emoji shortcodes, and common inline HTML such as `<u>`, `<mark>`, `<kbd>`, `<abbr>`, `<details>`) into readable terminal text outside fenced code blocks. Artifact previews strip common Markdown markers, and the shared TUI syntax style registers Prism-toned Markdown/code token colours.
- **Session records** now carry `uuid` and `parentUuid`, allowing append-only conversation forks; existing linear JSONL sessions receive deterministic implicit parents when loaded and remain resumable.
- **Escape follows the Claude Code prompt contract** — single empty Escape is a no-op, empty-input double Escape opens rewind, and non-empty double Escape saves then clears the draft; Ctrl+Q and the existing double Ctrl+C path remain explicit exit controls.
- **`AGENTS.md` and `CLAUDE.md`** expanded into full AI-collaborator entry points that link the repo's workflow, style, status, contribution, provider config, verification, and documentation-sweep rules.
- **Rapid-development workflow exception** treats `develop` as the active trunk for low-risk internal iteration while preserving PR/CR flow for high-risk or release-bound work.
- **OpenAI-compatible provider backend split** into request shaping, response parsing, streaming, wire types, and structured provider errors to harden the transport foundation.
- **Provider configuration requires the user-level `providers.yaml`** — Vesicle no longer falls back to a single `VESICLE_API_KEY` environment configuration when that file is missing. API keys load from the `.env` beside it, which takes precedence over inherited process variables so legacy project-root `.env` files cannot mask it; `vesicle doctor` reports whether that `.env` was found, without printing secret values.
- **Thinking-tier mapping** sends provider wire controls (`off` disables; `low`/`medium`/`high` map to high effort; `xhigh`/`max` map to max effort); unset sessions keep the provider/model default and send no thinking control fields. Generation defaults come from the selected model config instead of a hardcoded adapter temperature.
- **Provider configuration accepts `anthropic-messages`** (with optional `authMethod: x-api-key` or `bearer`) and **`gemini-generate-content`** (with `authMethod: x-goog-api-key`).
- **Shape-near singular/plural command pairs consolidated** — `/artifact` and `/engine` list without arguments and act with a target; redundant `/artifacts` and `/engines` are removed. `/effort` is now the thinking-control command, separated from `/reasoning` display; the old name and the hidden `/workflow` alias for `/engine` are removed, leaving `workflow` available for a future guided-workflow feature.
- **Public alpha documentation boundary declared**: setup, diagnostics, prompt shape inspection, and bundled examples are the supported onboarding path; comprehensive user documentation is intentionally deferred while feature and fix work remains the priority. The root README was reorganized into a concise installation and onboarding entry point with the detailed implementation inventory moved behind `STATUS.md`, natural line wrapping established as the Markdown prose convention, and synchronized Simplified Chinese editions of the README and contribution guide added.
- **npm/Bun package renamed** from the unowned scoped candidate `@prism/vesicle` to the available public package name `prism-vesicle`. The 1.0.0-alpha.1 release contract makes deterministic tests no longer run a real provider merely because local credentials exist, ships npm packages with only runtime files and pinned dependencies, resolves package assets and the OpenTUI worker independently of the caller's cwd, and adds `vesicle assets init` to create an editable project-local asset copy.

### Fixed

- **`source_materials/`** is now a writable project root for model-generated research and web captures; `/artifact` remains scoped to the four final-output roots.
- **`/artifact <n|path>`** renders the selected file in the message stream; the former implementation only wrote an unconsumed selection signal and claimed the preview was visible in a right-hand pane that had already been removed.
- **Sidebar artifact numbering** follows the fixed root order (`workspace`, `novels`, `reports`, `test_runs`) with newest files first inside each root, and the section scrolls within the remaining sidebar height instead of silently dropping entries after the first eight.
- **Non-confirm gate and question routes** type and paste into their always-visible Reject / freeform composer through one shared activation predicate across rendering, keyboard, and paste routing, preventing drift.
- **Non-confirmed `request_engine_switch`** decisions return their tool result to the provider and continue under the current engine; confirmed handoffs still switch only future turns without calling the provider under a new system prompt.
- **Slash-command candidate selection** renders reactively, resets when the filter query changes, and remains valid for empty result sets, so arrow keys move exactly one visible cursor.
- **Provider/model pickers** participate in bottom-layout sizing and cap visible rows to terminal height instead of overwriting the composer or telemetry footer; invalid `defaultModel` references are rejected while loading configuration, and `/model <model>` stays backward-compatible with active-provider switching.
- **Prompt contracts** align user-choice checkpoints with the current `ask_user_question` runtime, bind the Runtime engine's `runtime-turn` stop gate to `request_confirmation`, and keep RooCode-era mismatched tool names such as `ask_followup_questions` and `apply_diff` out of bundled prompts.
- **`vesicle prompt dump` / `prompt shape`** report the effective model-visible tool surface, including runtime-added `ask_user_question`, `request_engine_switch`, and stop-gate-only `request_confirmation`.
- **Windows Ctrl+C selection copying** writes clipboard text through a UTF-8 base64 bridge before `Set-Clipboard`, avoiding mojibake when copying Chinese or other non-ASCII text.
- **TUI tool calls and results** render as compact transcript summaries instead of dumping full tool arguments or full file contents into the main chat stream.
- **Streaming** rejects premature SSE EOF, reports malformed chunks with a provider-stream error, retries without OpenAI-specific `stream_options` for stricter compatible providers, and preserves the final assistant turn in the in-memory conversation history.
- **OpenAI-compatible streamed tool-call names** keep the latest provider value instead of concatenating repeated `function.name` deltas, avoiding duplicated names from non-conformant streams.
- **OpenAI-compatible reasoning responses** preserve provider `reasoning_content` through non-streaming, streaming, tool-loop follow-up requests, and session resume so reasoning models can use tools without losing required thinking context.
- **`/model` and session resume** resolve API-key availability through the user-level provider `.env`, so they no longer show "API key: missing" when the selected key is stored there.
- **Source and compiled startup** use separate OpenTUI setup paths: `bun run dev` preloads OpenTUI before importing the TUI, while `build:exe` applies the OpenTUI Bun build plugin explicitly, avoids external `bunfig.toml` preload lookup, and resolves compiled runtime files without changing the project cwd.
- **`bun run build:exe`** succeeds on Linux/macOS — the compile step emits the entry basename with a platform-appropriate extension, so the post-build rename no longer fails looking for a nonexistent `main.exe` and produces a host-native `prism-vesicle` binary outside Windows.
- **Standalone binaries preserve the launch directory as project root**; sessions, workspaces, and project asset overrides stay with the active project while runtime/default files resolve explicitly beside the executable. The embedded tree-sitter worker is now used even when the build directory's `node_modules/` remains reachable, preventing that development-only path from overriding the single-file runtime.
- **npm/Bun installs expose `vesicle` through an `.mjs` Bun launcher** that npm 11 retains during publication.
- **Rewind confirmation panels** with file changes reserve enough bottom layout height for the `Never mind` option, manual-edit warning, and footer. Inline freeform question composers and gate note/reject composers likewise reserve bounded layout rows to prevent option, input, and footer overlap in short terminals.
- **Composer-owned arrow keys** consume their original OpenTUI event before renderable dispatch, so cursor movement and prompt-history recall cannot also scroll the message stream or sidebar; modified arrows are consumed as composer no-ops when they have no composer action, and `Ctrl+C` consumes immediately while preserving its copy-selection / double-press exit contract.

## [0.1.0] - 2026-07-07

### Added

- Engine profile loader: `assets/engines/*.yaml` now drives systemPrompt composition, tool resolution, validator names, and declared stop gates at runtime. A hand-written YAML parser handles the narrow profile schema with no dependency. Profiles that name unknown tools or omit required fields fail loudly.
- Stop-gate runtime: the `request_confirmation` tool lets the model pause the workflow for user confirmation. The agent loop returns a `needs_user` result; `resolveGate()` feeds the decision back and continues. Undeclared gates are refused, not paused. ETL Phase 0 blueprint confirmation is wired end-to-end.
- Module A and Module B validators: v9 schema checks for character cards (frontmatter field allowlist, seven sections, Persona Topology subsections, Invariant/Variant axis counts, positive-shift direction, L-System leakage) and scenario cards (3–5 beat map, per-beat fields, tension range, non-monotonic trajectory, legacy field rejection).
- `vesicle prompt dump --engine <id>` and `vesicle prompt shape --engine <id>` print the fully composed system prompt and profile structure for "is there host pollution?" auditing.
- Session resume: `listSessions()` and `loadSessionMessages()` reconstruct prior turns. The TUI `/resume`, `/resume <n|id>`, `/new`, and `/help` commands manage sessions without hitting the provider.
- Markdown rendering in the TUI: assistant messages render through OpenTUI's `<markdown>` component with `conceal`, so headings, lists, emphasis, and code spans display as formatted output.
- Select-style gate UI borrowing Claude Code's PermissionPrompt shape: numbered Confirm/Reject options, Tab to expand confirm notes, and a persistent reject/discussion escape hatch.
- Centralised colour palette and shared syntax style in `src/tui/theme.ts`.

### Changed

- ETL stop gates now include `phase-confirmation` in addition to `blueprint-confirmation`, so Phase 1/2 artifact checkpoints use the `request_confirmation` tool instead of plain prose pauses.
- The TUI shell is now responsive: narrow terminals use a readable single message column, medium terminals add a workspace/artifact sidebar, and wide terminals add the activity/artifact pane.
- `/resume` now opens a TUI session picker, and sessions paused at an unresolved `request_confirmation` gate can be resumed back into the gate panel. Resumed sessions now load prior visible conversation into the message stream.
- ETL validators no longer run on ordinary assistant prose; they run only on artifact-shaped YAML-frontmatter assistant output.
- Wide TUI layouts now use the right pane for activity and recent artifacts instead of repeating the last assistant output.
- The agent loop emits coarse activity events for provider requests, assistant responses, tool calls, gate pauses, and validation.
- The input area now shows slash-command hints and supports Up/Down prompt history recall.
- Agent-loop tool ceiling raised 6 → 40 with a no-progress circuit breaker (4 consecutive failing-tool rounds stop the loop and persist the last response instead of throwing). Vesicle's controlled file tools do not need the strict caps a coding-agent host would impose.
- `RunPromptResult` is now a discriminated union (`complete` | `needs_user`) with an optional `validation` outcome on the complete branch.
- ETL engine prompt Phase 0 now instructs an explicit `request_confirmation` call instead of prose "wait for user confirmation"; a new Stop Gate contract section documents all gated vs conversation-bound points.
- Validator outcomes surface in the TUI message stream and session log; failures are advisory and never abort a turn.

### Fixed

- Gate confirmation now renders in a dedicated bottom panel with bounded summary text, stable option rows, and no side-panel squeezing at gate time.
- Runtime artifact roots (`workspace/`, `test_runs/`, `novels/`, `reports/`, `source_materials/`) are now gitignored with `.gitkeep` stubs, so model/user output does not enter git history.
