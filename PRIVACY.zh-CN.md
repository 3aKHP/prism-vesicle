# 隐私政策

[English](./PRIVACY.md) | [简体中文](./PRIVACY.zh-CN.md)

最后更新：2026-07-21

## 摘要

Prism Vesicle 是一个在用户本机运行的开源终端应用。Prism Vesicle 项目不运营遥测、分析、广告、崩溃上报、用户账户或内容上传服务。应用的工作数据存放在用户自己的计算机上；只有当用户配置或调用确实需要外部服务的功能时，应用才会连接这些服务。

这些外部服务各自有独立的条款、保留规则和隐私政策。除非用户确认所选服务及账户协议适合处理机密、个人或受监管材料，否则不应通过它们处理此类内容。

## 本地保存的数据

根据实际使用的功能，Prism Vesicle 可能在本地保存：

- 用户配置目录中的供应商、模型、Tavily、MCP、权限与资产设置，以及同目录 `.env` 密钥文件中的 API key；
- 当前项目 `.vesicle/sessions/` 下的对话、模型响应、工具记录、用量元数据、确认门和引擎切换记录；
- `.vesicle/attachments/` 下的图片附件副本；
- `.vesicle/file-history/` 下的文件回退历史；
- `.vesicle/processes/` 下有长度限制的后台进程输出与状态；
- 作为本地会话状态保存的 SubAgent 状态和投递记录；
- 用户所选项目目录中的导入资料与生成文件。

默认用户配置目录在 Windows 上是 `%APPDATA%\prism-vesicle`；设置了 `XDG_CONFIG_HOME` 时是 `$XDG_CONFIG_HOME/prism-vesicle`；其他情况下是 `~/.config/prism-vesicle`。部分配置路径可以通过环境变量覆盖。

仅仅安装或运行 Prism Vesicle，并不会把这些本地数据发送给项目维护者。

## 发送给模型供应商的数据

用户发起模型请求时，Prism Vesicle 会把数据发送给用户配置中选择的供应商和 endpoint。根据具体请求及当前模型能力，发送内容可能包括：

- 当前引擎的 system prompt 与模型可见工具定义；
- 当前对话上下文与用户提示；
- 用户主动附加到对话的图片；
- 继续本回合所需的相关工具结果和生成上下文；
- 生成参数和常规协议元数据。

引导式 Setup 可能把配置的 API key 发送给所选供应商，并调用其 `/models` endpoint 来发现可用模型 id。该 key 用于向用户选择的服务进行身份验证。供应商自己的隐私、日志、训练、数据地域和保留条款适用。

## 可选外部服务

### Tavily

配置 Tavily 后，只有当用户或获准的模型操作调用 Web 工具时，Tavily 才会收到完成操作所需的搜索词、URL、crawl/map 请求或 research 请求，以及身份验证和常规请求元数据。仅仅保存 Tavily key 不会自动联系 Tavily。

### MCP 服务器

启用 MCP 服务器后，Prism Vesicle 可以向它发送初始化请求、工具清单请求、配置的 HTTP headers、工具名和工具参数。只有在调用其工具时，服务器才会收到相应工具调用，但 Setup 和诊断功能可能连接服务器以进行测试或列举工具。服务器运营者的隐私政策适用。配置的 headers 可能包含从本地 `.env` 展开的凭据。

### 用户授权的命令与 URL

`shell_exec` 在本机执行获准的命令，它不是 Prism 托管的服务。由用户批准或当前权限模式允许的命令、脚本或程序，可以自行读取数据或访问网络。同样，通过 Web 工具访问 URL 时，目标网站和相关服务提供方会获得正常网络连接信息以及所请求的 URL。

## 密钥

API key 与 header 密钥应保存在用户级 `.env` 或进程环境变量中。Prism Vesicle 不会有意把它们写入会话或项目配置文件。只有在向用户明确配置的外部服务进行身份验证时，相关密钥才会被发送。Prism Vesicle 无法控制该外部服务自行进行的日志记录或数据保留。

用户不应把密钥粘贴到提示、截图、issue、日志或 Git 跟踪的项目文件中。密钥一旦泄露，应立即在对应供应商处吊销并更换。

## 安装与卸载

Windows 安装器会为当前用户安装程序文件，把安装目录加入该用户的 `PATH`，创建开始菜单入口，注册资源管理器目录操作，并注册卸载程序。修复或升级可能恢复或替换这些组件。

普通卸载会移除安装器负责的程序文件和 Windows 集成。为了避免卸载或升级破坏用户工作，它会有意保留用户配置目录、凭据、本地项目目录、生成内容以及项目中的 `.vesicle/` 状态。

## 用户控制与删除

用户可以控制配置哪些供应商和可选服务、打开哪个项目、批准哪些工具以及提交什么内容。若要阻止今后的外部传输，请禁用或删除对应的供应商、Tavily 或 MCP 配置，并在服务提供方处吊销其凭据。

关闭 Prism Vesicle 后，可以按以下方式删除本地数据：

- 删除对应项目的 `.vesicle/` 目录，以移除该项目的 Vesicle 会话、附件、历史、进程与 SubAgent 状态；
- 删除不再需要的生成文件或导入资料；
- 删除 Prism Vesicle 用户配置目录，以移除保存的供应商、MCP、权限、资产与密钥配置；
- 对于已经传输给外部服务的数据，使用该服务自己的账户工具或支持渠道处理。

除非用户另有备份，否则删除 `.vesicle/`、生成文件或用户配置目录后无法恢复。

## 变更与问题

本政策如有实质变化，会通过公开 commit 更新，并保留在项目历史中。不包含个人数据、凭据、私有供应商 URL 或机密内容的问题，可以提交到项目的 [GitHub issues](https://github.com/3aKHP/prism-vesicle/issues)。

发布完整性和签名文件处理方式见[代码签名政策](./CODE_SIGNING_POLICY.zh-CN.md)。
