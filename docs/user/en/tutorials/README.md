# Tutorials

English | [简体中文](../../zh-CN/tutorials/README.md)

Every [start page](../README.md) converges on the same point: `vesicle doctor` passes and Vesicle is open in your project directory. From here you learn how to actually create with it.

The tutorials assume you can already start Vesicle and that `vesicle doctor` reports `Missing: none`. Each one introduces only what its task needs and ends with a checklist.

## Learning path

1. [First conversation: interface and gates](./first-conversation.md) — meet the interface and understand "gates", the pause-to-confirm interaction.
2. [Keep working during a turn](./work-while-running.md) — add to the main task through the queue and ask temporary side questions with `/btw`.
3. [First character card](./first-character-card.md) — run ETL Workflow A end to end and produce a Module A character card that passes validation.
4. [First scenario card](./first-scenario-card.md) — produce a Module B scenario card from a character card and validate it.
5. [Sessions and rewind](./sessions-and-rewind.md) — resume past sessions and rewind to any step.
6. [Set up Persistent Instructions](./persistent-instructions.md) — maintain cross-session project rules with `/init`, `/instructions`, and model tools.
7. [Permissions and the host shell](./permissions-and-shell.md) — the four tool-approval modes and when to enable shell.

After these seven you have the daily creation loop. For a command cheatsheet, configuration, the security model, and more, see the [reference section](../reference/README.md).
