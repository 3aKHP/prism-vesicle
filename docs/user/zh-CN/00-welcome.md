## 本章目标

你将了解 Prism Vesicle 能做什么、本手册会要求你完成哪些准备，以及安装任何内容之前必须理解的安全规则。

**预计耗时：** 10 分钟

**前置条件：** 无

## Prism Vesicle 是什么

Prism Vesicle 是一个通过 AI 模型供应商运行 Prism Engine 提示的终端应用。你在文本界面中输入指令，所选模型作出响应，而 Vesicle 可以保存对话、请求确认、使用经过批准的工具，并在受保护的目录中创建项目文件。

Vesicle 是宿主应用，本身不包含 AI 模型。要进行真实对话，你需要用自己的 API 密钥连接受支持的模型供应商。

默认的 Prism 引擎是 ETL，它可以帮助你把创意和研究材料转换成结构化的角色与情景资料。后续章节会介绍其他引擎和更高级的工作流。

## Prism Vesicle 不是什么

- 它不是网页，也不是传统的图形化桌面应用。
- 它不是 AI 订阅服务，也不包含模型调用额度。
- 它不是可以无限制访问计算机的通用编码代理。
- 它不能保证模型输出一定正确、安全或适合发布。

## 你将配置的四项内容

1. 一个可以输入命令的终端。
2. 用于启动 Vesicle 的 Bun 运行时。
3. 一个用于保存工作的项目文件夹。
4. 存放在 Windows 用户配置中的模型供应商设置与 API 密钥。

入门路径不需要 Git、WSL、Linux、Visual Studio 或管理员权限。

## 安全规则

### 保护 API 密钥

API 密钥类似于一个允许软件消耗模型供应商账户额度的密码。绝对不要把它粘贴到聊天消息、截图、Issue、`providers.yaml` 或项目内的文件中。

本手册会把密钥存放在 `%APPDATA%\prism-vesicle\.env`。第 04 章会给出准确步骤。

### 了解费用

大多数 API 供应商会按模型用量收费。创建密钥前，请阅读供应商当前的价格和账单控制方式。如果供应商支持，请从少量余额或支出上限开始。

### 检查生成文件

AI 输出可能包含事实错误、不需要的内容或结构问题。分享或发布重要制品前，应先进行预览和验证。

Vesicle 可以在受保护的项目根目录下使用嵌套目录组织生成内容。它能够创建和查看目录、移动或重命名目录树，并删除空目录，但不能删除固定根目录或穿越符号链接。

### 不要以管理员身份运行

入门步骤使用普通 Windows 账户。如果 Windows Terminal 的标题中出现 `Administrator` 或“管理员”，请关闭它并重新打开普通终端。

## 本手册如何展示命令

命令会显示在这样的代码块中：

```powershell
bun --version
```

只复制命令文字，将其粘贴到 PowerShell，然后按 Enter。不要额外输入终端提示符 `>`。一个代码块中有多行命令时，除非正文明确要求整体粘贴，否则请从上到下逐行执行。

`YOUR_API_KEY` 之类的文字是占位符，需要替换成对应的真实内容。反引号中的路径、命令名和配置字段名通常应保持原样。
