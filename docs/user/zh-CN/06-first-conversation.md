# 06 — 完成第一次对话

[← 上一章：Doctor 与启动](./05-doctor-and-launch.md) | [手册目录](./README.md) | [English](../en/06-first-conversation.md)

## 本章目标

你将发送一个安全的首次提示，识别模型请求的各个阶段，使用输入框的基本操作，并确认对话会自动保存。

**预计耗时：** 15 分钟

**前置条件：** 第 00–05 章，并且 Vesicle 当前处于打开状态

## 识别主要界面区域

Vesicle 可能根据终端宽度重新排列或隐藏次要信息。重要区域包括：

- 显示用户与助手消息的对话流
- 显示当前供应商、模型、引擎和活动状态的状态栏或底栏
- 位于底部、用于输入文字的输入框
- 可选的制品、MCP 工具或会话状态侧栏信息

窗口较窄时，布局可能变得更简洁，但底层功能不会因此消失。

## 发送安全的首次提示

必要时先点击终端，在底部输入框中输入下面的提示，然后按 Enter：

```text
在开始项目之前，请用简单易懂的语言解释当前 Prism ETL 引擎可以帮助我创建什么。暂时不要创建或编辑任何文件。
```

Vesicle 会把请求发送给已配置的供应商。处理期间，状态文字会发生变化，响应内容可能逐步显示。

准确答案不是预先固定的，因为它来自所选模型。只要该回合最终显示助手响应，而不是供应商或密钥错误，就表示成功。

## 理解刚才发生了什么

在这个回合中，Vesicle：

1. 加载了当前 ETL 引擎配置和提示资产
2. 将它们与你的消息和可用工具定义组合
3. 把请求发送给供应商
4. 以流式或完整方式显示响应
5. 把该回合追加到 `.vesicle/sessions/` 下的当前会话

“不要创建文件”的要求让第一次交互更容易检查。后续工作流章节会有意允许创建制品。

## 练习输入框操作

开始输入第二条消息，但先不要立即提交。

- Backspace 和 Delete 用于编辑草稿。
- Ctrl+Enter 在草稿内插入换行。
- Enter 提交完整草稿。
- 上、下方向键会先在多行草稿内移动，之后才会调出历史提示。
- 模型请求正在进行时，Escape 会取消该请求。

作为练习，使用 Ctrl+Enter 在两行之间换行，输入：

```text
请举出三种有助于 ETL 项目的源材料。
每个例子只用一句话，并且不要创建文件。
```

按 Enter 提交。

## Vesicle 提问或请求确认时

Prism 引擎可能暂停并等待用户输入。面板可能提供多个选项、开放回答、Confirm 或 Reject。

- 使用方向键在选项之间移动。
- 按当前面板显示的提示提交选择。
- Confirm 表示工作流可以通过面板描述的检查点继续。
- Reject 会把反馈返回模型，并不表示应用运行失败。

首次解释性提示通常不需要文件确认，但模型行为可能不同。选择前请阅读面板摘要。

## 在不调用供应商的情况下查看上下文

输入并提交：

```text
/context
```

这是 Vesicle 本地命令。它会报告可用上下文和用量信息，不会再次发送模型请求。只有配置的模型声明了上下文限制时，部分百分比才会显示。

## 确认会话持久化

按 Ctrl+Q 退出 Vesicle，然后在同一个项目文件夹中再次运行 `vesicle .`。此过程不依赖任何全局项目指针。

Vesicle 把会话文件保存在项目的 `.vesicle` 目录中。后续手册章节会详细介绍 `/resume`、会话选择和创建全新会话。

## 完成检查

满足以下条件时，你已经完成入门路径：

- 至少有一次模型响应成功完成
- 你使用过 Enter 提交，并使用 Ctrl+Enter 创建多行草稿
- 你理解 Escape 会取消正在进行的请求
- `/context` 生成了本地状态消息
- 你知道会话保存在项目中，而不是供应商配置文件夹中

现在你已经可以使用 Vesicle 进行探索性对话。在后续章节完成前，如果重要工作涉及文件生成、引擎移交、回退、外部研究或 MCP 工具，请继续遵循本手册的指导。
