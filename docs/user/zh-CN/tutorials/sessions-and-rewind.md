# 会话恢复与回退

[English](../../en/tutorials/sessions-and-rewind.md) | 简体中文

Vesicle 把每一段对话持久化在项目里,所以你可以随时回到旧会话,也可以在当前会话里**回退**到任意一步重写。这篇讲两件事:跨会话的恢复,和单会话内的回退。

## 会话存在哪里

每个项目目录下有一个 `.vesicle/`:

- `.vesicle/sessions/` —— 会话记录(append-only 的 JSONL),包含每一轮的用户输入、模型回复、工具调用、门决策、用量等。
- `.vesicle/file-history/` —— Vesicle 工具改动过的文件的检查点(每个真实用户回合一份)。

会话属于项目,跟着项目目录走;换一台机器、换一个项目目录,就是不同的会话集合。

## 恢复旧会话

```
/resume
```

弹出本项目的历史会话列表(按时间),选一个恢复。也可以直接 `/resume 2` 或 `/resume <会话 id>`。恢复时会还原对话、未决的门、模型选择等。

想从零开始而不是恢复:

```
/new
```

## 回退:回到任意一步

`/rewind`(别名 `/checkpoint`)打开一个**回退选择器**,列出你这次会话里发过的每一条 prompt。选一条:

- 对话会**还原到那条 prompt 之前**的状态,并把那条 prompt 重新填回输入框,方便你改完再发。
- 旧的那条分支**不会被删除**,你可以放心试探。
- 你还能选择是否一起还原那一回合里 Vesicle 改过的文件;确认前它会列出会变化的文件和增删行数。

快捷键:输入框**空着时连按两次 Esc**(800 毫秒内)直接打开回退选择器。

> 回退的文件检查点只覆盖 Vesicle 自己工具改过的文件。你在 Vesicle 之外手动改的文件不在这个账本里,不会被当作回退目标。

## 上下文太长时:压缩

对话变长后,可以把它压成一段摘要再继续,省下上下文:

```
/compact
```

可选地附上说明,例如 `/compact 重点保留角色卡的拓扑决策`。压缩会通过当前模型生成摘要,然后从摘要接着聊;原文仍留在会话记录里。

切引擎时也能顺手压缩:`/engine <id> --summary`。

## 退出与中断

- Ctrl+Q —— 退出 Vesicle(会话已经持久化,下次 `/resume` 能找回)。
- Esc —— 中断正在跑的请求(不会丢已写入的文件)。
- 输入框有内容时连按两次 Esc —— 存草稿并清空(不发送)。

## 检查点

- [ ] 你用 `/resume` 恢复过一个旧会话。
- [ ] 你用 `/rewind` 回退到某一步并重新发了一条 prompt。
- [ ] 你知道 `.vesicle/sessions/` 和 `.vesicle/file-history/` 各存什么。
