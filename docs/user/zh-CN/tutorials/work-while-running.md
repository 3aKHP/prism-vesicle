# 运行中继续工作:队列与旁路提问

[English](../../en/tutorials/work-while-running.md) | 简体中文

一个创作回合可能包含多次供应商请求和工具调用。你不必等整个回合结束才补充要求,也不必为了问一句背景问题而打断主任务。这篇教程会实际使用输入队列和 `/btw`。

## 在运行中的回合追加要求

先发起一个需要读取素材并形成方案的任务。状态栏显示 Agent Loop 正在运行时,继续在输入框输入:

> 角色的公开目标不要提前暴露真实动机。

按 Enter 后,这条消息会出现在输入框上方的 FIFO 队列中。Vesicle 会让当前工具轮次完整结束,再在下一次供应商请求前把消息交给主对话。如果主回合先结束,它会立即作为下一条输入处理。

再排入一条消息,然后在输入框为空时按 Up。最新的队列输入会回到输入框,你可以修改后重新提交。队列按提交顺序处理;会切换或重置会话的操作会清空当前队列。

如果新要求必须立刻取代正在执行的方向,按 Esc。Vesicle 会中断当前供应商请求或工具操作,随后优先处理下一条队列输入。不要把 Esc 当成普通的“加急”:中断可能让本轮尚未完成。

## 在主回合旁边问一句

主回合仍在运行时输入:

```text
/btw 当前方案里已经确定了哪些不可变条件?
```

答案会在临时覆盖层里流式显示,主回合继续在下方运行。这个问题只读取主对话最近一个完整、可发送给供应商的上下文边界;它不会看到半写完的工具轮次,也不会进入主对话影响后续创作。

覆盖层中可以使用:

- Up / Down:滚动当前答案。
- Left / Right:切换本次进程里的旁路问答。
- `c`:复制当前答案的 Markdown。
- `x`:清空当前会话的旁路问答并关闭。
- Esc:关闭覆盖层;如果答案仍在生成,只取消旁路请求,不取消主回合。

关闭后输入不带问题的 `/btw`,可以重新打开最近一次答案。

## 选择正确的入口

- **队列消息**会进入主对话,适合补充约束、纠正方向或安排下一步。
- **`/btw`** 不进入主对话,适合查一句背景、回顾当前条件或解释术语。
- `/btw` 不声明工具,不能替你读取新文件、修改制品或执行命令;旁路记录只在当前进程内存在,重启后消失。

Slash 命令在忙碌回合中的调度并不相同:纯宿主查询可以立即执行,制品读取会等当前工具轮次,配置、选择器和会话命令会等整个 Agent Loop。需要查具体命令时看[命令速查](../reference/commands.md)。

## 检查点

- [ ] 你在运行中的回合排入过一条补充要求。
- [ ] 你用 Up 取回并编辑过最新队列输入。
- [ ] 你用 `/btw` 提问,并在不打断主回合的情况下关闭了答案。
- [ ] 你能区分“写入主对话的队列消息”和“临时的旁路问题”。
