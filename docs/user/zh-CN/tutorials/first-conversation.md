# 第一次对话:界面与门

[English](../../en/tutorials/first-conversation.md) | 简体中文

这篇带你认识 Vesicle 的界面,并理解它最核心的交互——**门**(gate):引擎在关键节点会暂停,等你确认后才继续。学完你能看懂一次完整的对话回合。

## 准备一点素材

Vesicle 默认用 ETL 引擎,它把你的素材加工成角色卡、情景卡。先在项目里放一条素材:

```bash
mkdir -p source_materials
cat > source_materials/note.md <<'EOF'
# 角色速记:林越
28 岁,前战地记者,现在开一间深夜咖啡馆。话少,观察力极强。
左腕有一道旧疤。习惯把重要的事写在纸杯上。
EOF
```

## 启动并发出第一条

确保在项目目录里启动:

```bash
vesicle .
```

界面分几块:**上方是对话区**,侧边是**工作区**(产物列表),**底部是输入框**。

在输入框里写你的意图,回车发送:

> 读取 source_materials/note.md,基于林越这个人物给我一个角色概念草案。

输入框约定:

- 回车 = 发送。
- Ctrl+Enter = 换行(想写多行时用)。
- Esc = 中断正在跑的请求。

引擎会读取素材,然后在对话区给出一个**蓝图**(Target Concept、Archetype、Core Desire 等)。这一步不写文件。

## 门:暂停等你拍板

蓝图给出后,引擎**不会**自己往下写——底部会弹出一个确认面板,通常是**确认 / 拒绝**两个选项。这就是"门"。

- 选**确认** —— 引擎进入下一阶段。
- 选**拒绝**(留空拒绝也可以)—— 引擎会问你哪里要改,改完再来。

> 门是 Vesicle 的常态:蓝图、每个写作阶段、某些工具调用前都会停下来等你。它的意义是**让你在每个关键节点保持控制权**,而不是让模型一口气写到底。

## 看懂底部状态栏

发完一轮,底部会显示这一轮的用量,形如 `↑1.2k ↓340 ↻0`(上行 / 下行 / 缓存 token)和上下文占用百分比。想看详情用:

```
/context
```

## 常用命令速览

在输入框以 `/` 开头的是命令,敲 `/` 会弹出候选菜单(↑↓ 选择、Tab/Enter 补全):

- `/help` —— 列出全部命令。
- `/context` —— 当前上下文用量。
- `/artifact` —— 列出已生成的产物。
- `/engine` —— 查看当前引擎(默认是 ETL)。
- Ctrl+Q —— 退出 Vesicle。

## 检查点

- [ ] 你发出了一条 prompt 并收到了蓝图。
- [ ] 你在门上做过一次确认或拒绝。
- [ ] 你用 `/context` 看过用量。
