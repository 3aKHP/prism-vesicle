# 02 — 模型供应商、API 密钥、费用与隐私

[← 上一章：Windows 基础](./01-windows-basics.md) | [手册目录](./README.md) | [English](../en/02-model-providers.md) | [下一章：安装 →](./03-installation.md)

## 本章目标

你将了解 Vesicle 所需的服务账户、创建或找到 API 密钥，并在连接程序前对费用和隐私作出知情选择。

**预计耗时：** 15–30 分钟，具体取决于供应商注册流程

**前置条件：** 第 00–01 章，以及一个受支持模型供应商的账户访问权限

## 模型供应商做什么

模型供应商负责运行远程 AI 服务。Vesicle 会把对话和经过允许的工具结果发送给该服务，然后显示模型返回的响应。

当前 Vesicle 版本支持以下协议系列：

- OpenAI-compatible Chat Completions
- Anthropic Messages
- Gemini `generateContent`

入门路径使用 DeepSeek，因为仓库当前的示例配置已经包含它。这只是教程选择，不代表产品背书或强制要求。高级章节会介绍 Anthropic、Gemini、本地兼容服务器和多供应商配置。

## 创建供应商账户

在浏览器中打开供应商官方网站并创建账户。对于本教程使用的配置，请打开 [DeepSeek Platform](https://platform.deepseek.com/)，并通过[官方 API 文档](https://api-docs.deepseek.com/)确认当前模型 id、API 密钥和账单信息。

供应商网站会独立于 Vesicle 发生变化，因此本手册不会逐个描述按钮。输入付款信息或创建密钥前，请确认你正在访问供应商的真实域名。

## 检查价格与限制

使用 API 密钥前：

1. 阅读计划使用模型的当前价格。
2. 确认账户是否需要预付余额或自动扣款。
3. 如果供应商支持，设置较小的支出上限或提醒。
4. 理解长对话、大型源文件、工具循环和推理模型可能消耗更多 token。

当供应商返回用量数据时，Vesicle 会显示相关信息，但实际费用始终以供应商账单页面为准。

## 创建 API 密钥

在供应商控制台中创建新的 API 密钥。密钥通常是一段很长的随机字符。

暂时将其复制到安全的密码管理器，或保持供应商页面开启直到第 04 章。不要把密钥粘贴到本手册、聊天对话、截图或 `providers.yaml` 中。

如果密钥意外暴露，请在供应商控制台中撤销它并创建替代密钥。

## 理解哪些内容会发送给供应商

执行模型请求时，供应商可能收到：

- 当前 Prism 引擎的系统提示
- 当前回合所需的对话上下文
- 你附加的文本或图像
- 模型可见的工具定义
- 当前回合返回的工具结果

Vesicle 会让仅供宿主使用的会话元数据和密钥配置避开正常供应商消息，但除非供应商及账户策略适合处理机密材料，否则仍不应提交这些内容。

## 选择教程模型

仓库当前示例把 `deepseek-v4-flash` 设为默认教程模型。供应商的模型目录可能变化。如果供应商控制台或 API 文档已经不再提供这个准确 id，请选择当前支持工具调用的 DeepSeek 模型，并在第 04 章使用其准确 API 模型 id。

不要根据宣传名称猜测模型 id。请从供应商当前 API 文档复制准确的模型 id。

## 完成检查

继续前请确认：

- 你拥有供应商账户
- 你理解该账户如何计费
- 你已经创建 API 密钥，或知道在哪里创建
- 你知道供应商 Base URL；供应商支持 `/v1/models` 时，Setup 会自动发现模型 id
- 你没有把 API 密钥放进项目文件夹
